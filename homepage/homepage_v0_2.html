// principles

Built different.

01

No Build Process

Everything is pre-compiled and ready. Just include the stylesheet and start building. No bundlers, no watchers, no waiting.

02

Value-Driven

Generic utility names paired with CSS variables you define. Write class="m" and style="--m: 4". No arbitrary scales.

03

Runtime States

Hover, focus, and other interactive states are generated at runtime—keeping your CSS bundle lean and performant.

04

Responsive

Familiar breakpoint prefixes like sm:, md:, and lg: for building interfaces that work on any screen size.

05

Theme-Aware

Built-in dark mode support with the dark: prefix. Seamless theming without configuration overhead.

06

Explicit Control

No magic. Utilities and values always go together intentionally. You know exactly what each class does.

// quick start

See how it works.

example.html 
<!-- Value-driven utilities -->
<div
  class="m sm:m md:m"
  style="--m: 4; --sm-m: 8; --md-m: 16"
>
  Responsive margins
</div>

<!-- Interactive states -->
<button
  class="bg bg:hover px py"
  style="--bg: var(--uk-primary); --bg-hover: #27272a; --px: 5; --py: 2.5"
  data-uk
>
  Hover me
</button>

Get started in seconds.

Recommended

CDN

Drop this into your HTML head:

<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/frankenstyle@0.1.0-next.11/dist/css/frankenstyle-kit.min.css">

npm

Install with your package manager:

npm install frankenstyle
